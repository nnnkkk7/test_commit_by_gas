<h3 id="behavioral_changes_2" data-text="Behavioral Changes">Behavioral Changes</h3>

<dl>
<dt>Review IDs</dt>
<dd>Review IDs have migrated to a new format. This new ID will now be returned in all applicable API responses. For a short period of time, old review IDs will still resolve when used to call the API, but it is critical to refresh all review IDs stored on your system within 30 days as required by our <a href="/my-business/content/policies#content-storage">policies</a> by calling the <a href="/my-business/reference/rest/v4/accounts.locations.reviews/list">listReviews</a> endpoint.</dd>
<dt>Service Area Business</dt>
<dd>In Q1 2021, <code translate="no" dir="ltr">Address</code> will no longer be returned in the <a href="/my-business/content/my-business/reference/rest/v4/accounts.locations#resource:-location">Location</a> object for <a href="/my-business/reference/rest/v4/accounts.locations#Location.BusinessType"><code translate="no" dir="ltr">CUSTOMER_LOCATION_ONLY</code></a> businesses.</dd>
</dl>

<h3 id="backward-incompatible_changes" data-text="Backward-incompatible changes">Backward-incompatible changes</h3>

<ul>
<li><p>Photos, previously a field of <code translate="no" dir="ltr">location</code> objects, are now a
resource under location (<code translate="no" dir="ltr">accounts/<var translate="no">account_ID</var>/locations/<var translate="no">location_ID</var>/photos</code>).</p></li>
<li><p>Address field <code translate="no" dir="ltr">Country</code> is now <code translate="no" dir="ltr">region_code</code> and
<code translate="no" dir="ltr">sub_locality</code> is now <code translate="no" dir="ltr">sublocality</code>.</p></li>
<li><p>Previously deprecated <code translate="no" dir="ltr">Ownership</code> field removed.</p></li>
<li><p>Previously co-owners of accounts and locations were indistinguishable from
primary owners through the API (all appeared as <code translate="no" dir="ltr">OWNER</code>). Now primary
owners will be given the <code translate="no" dir="ltr">OWNER</code> role and other owners will be given the
<code translate="no" dir="ltr">CO_OWNER</code> role.</p></li>
<li><p><code translate="no" dir="ltr">AccountType.BUSINESS</code> is now named <code translate="no" dir="ltr">AccountType.LOCATION_GROUP</code>.</p></li>
<li><p>When using a field mask for updating locations, provide <code translate="no" dir="ltr">update_mask</code> in
place of the existing <code translate="no" dir="ltr">field_mask</code>.</p></li>
<li><p>The category message used for <code translate="no" dir="ltr">primary_category</code> and <code translate="no" dir="ltr">additional_categories</code>
no longer contains the human-readable form. Users should use the categories
service of the Google My Business API to get translated, human-readable category names.</p></li>
</ul>
